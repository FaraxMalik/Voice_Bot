AI-Powered Voice Agent

A fully local, end-to-end pipeline that listens to a caller, understands their intent using a large language model, and responds with natural speech — replacing traditional call agents with no cloud dependency and zero licensing cost.

March 2026  |  100% Open Source  |  Runs Locally — No Cloud Required
Section 01

What We Are Building

When a user speaks, their voice is transcribed to text by a Speech-to-Text model. That text is passed to a locally running large language model (LLaMA) which generates a context-aware reply. The reply is then converted back to natural-sounding speech using a custom Text-to-Speech model fine-tuned specifically for this project.

The final audio is played back to the user — completing one full turn of a voice conversation without any human agent involvement and without any paid cloud service. All processing happens on a single machine. A FastAPI server exposes clean REST endpoints so any frontend or telephony interface can connect to it.

Section 03

Component Breakdown

Role Model / Tool Key Details Cost
Speech-to-Text faster-whisper (small, 74M params) Optimized Whisper runtime, 4x faster than original on CPU. Returns clean punctuated transcript. Handles short conversational clips in 2–4 seconds. Free (MIT)
Language Model LLaMA 3.2 3B — Q4_K_M quantized GGUF via Ollama Meta's latest small model. Q4 quantization keeps RAM usage at ~2 GB. Generates 8–15 tokens/sec on CPU. Maintains full conversation history for context. System prompt configures it to behave as a call agent. Free (Meta License)
Text-to-Speech Custom XTTS v2 — fine-tuned on LJSpeech 1.1 Built on Coqui XTTS v2 base. Fine-tuned on 13,100 audio clips (~24 hours). GPT-2 backbone (441M parameters), HiFi-GAN neural vocoder. DVAE encoder frozen from base model. Supports voice cloning via a short reference clip. Free (CPML)
API Backend FastAPI + Uvicorn (Python) Hosts all three models. Exposes endpoints: /voice/process (full pipeline), /voice/transcribe, /voice/chat, /voice/speak. Models load once on startup. Free (MIT)
Section 04

Reducing Response Time Per Query

Running all three models sequentially on CPU produces a total latency of 20–40 seconds per turn in the naive case. The following optimizations bring the time-to-first-audio down to approximately 6–12 seconds.

Step Naive Time (CPU) After Optimization
Speech-to-Text (5s audio clip) 2–4 seconds 1–2 seconds (switch to tiny model)
LLM generation (50 token reply) 8–15 seconds total First sentence ready in ~3 seconds (streaming)
TTS synthesis 8–20 seconds Starts before LLM finishes (streaming)
Total per turn 20–40 seconds 6–12 seconds to first audio chunk
